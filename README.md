Game Management System - Backend Task
מערכת בקאנד מבוססת Node.js לניהול משחקים ומשתתפים, המשתמשת ב-Prisma ORM לעבודה מול מסד נתונים PostgreSQL. הפרויקט ארוז ב-Docker להרצה מהירה ונוחה.

🚀 תכונות עיקריות
מודל נתונים רלציוני: ניהול קשרים של Many-to-Many בין משתמשים למשחקים באמצעות טבלת משתתפים.

לוגיקה עסקית: שירות הצטרפות למשחק הכולל ולידציות על סטטוס המשחק ומניעת כפל רישום.

סביבת פיתוח מבודדת: הגדרת Docker מלאה הכוללת Healthcheck למסד הנתונים כדי להבטיח הרצה חלקה.

Automated Seeding: המערכת יוצרת נתוני דמי (משתמש ומשחק) באופן אוטומטי בעת ההרצה הראשונה לצורך הדגמה.

🛠 טכנולוגיות
Runtime: Node.js 18.

ORM: Prisma עם Prisma Client.

Database: PostgreSQL 15.

Infrastructure: Docker & Docker Compose.

🚦 הוראות הרצה
כדי להריץ את הפרויקט, ודא ש-Docker מותקן והרצ את הפקודה הבאה מהתיקייה הראשית:

Bash
docker-compose up --build
מה הפקודה מבצעת?

מרימה מסד נתונים PostgreSQL.

מבצעת Build ל-Dockerfile ומתקינה תלויות.

מריצה prisma db push לסנכרון הסכימה מול ה-DB.

מפעילה את האפליקציה שמבצעת חיבור, יצירת נתונים והצטרפות למשחק.

📂 מבנה הפרויקט
schema.prisma: הגדרת המודלים (User, Game, Participant) והקשרים ביניהם.

game.service.js: פונקציות הליבה של ניהול המשחק.

main.js: נקודת הכניסה לאפליקציה המדגימה את זרימת העבודה.

docker-compose.yml: ניהול הקונטיינרים ומשתני הסביבה.

הערה: הוגדרו מעקפי SSL ב-Dockerfile לצורך תאימות בסביבות עבודה מסוננות.
